## 2. Thinky의 Inkling: 975B-A41B 멀티모달, 새로운 미국산 Apache 2.0 오픈 모델 및 Inkling-Small\(276B-A12B\)

- **출처:** Latent Space
- **게시:** 2026.07.16 15:18
- **중요도:** 82
- **읽기 시간:** 18분

### 요약

Thinking Machines Lab이 텍스트·이미지·오디오를 처리하는 open-weights MoE 모델 Inkling을 공개했다. Inkling은 총 975B, 토큰당 41B 활성 파라미터를 사용하고 최대 1M 토큰 컨텍스트를 지원하며, Apache 2.0 라이선스로 제공된다. 12B 활성 파라미터를 사용하는 경량형 Inkling-Small도 함께 소개됐고, Hugging Face와 Tinker 및 여러 추론 인프라에서 지원된다.

### 왜 중요한가

Inkling은 미국 기업이 공개한 대규모 멀티모달 open-weights 기반 모델로, 직접 호스팅·파인튜닝·후속 RL에 활용할 수 있는 선택지를 넓힌다. 최고 벤치마크 모델을 목표로 하기보다 효율적인 추론, 조절 가능한 reasoning effort, 커스터마이징 가능한 기반 모델을 강조한다는 점이 AI-assisted software development와 생산성 도구 구축에 의미가 있다.

### 개발자 영향

개발자는 Apache 2.0 조건 아래 Inkling을 자체 인프라나 vLLM·SGLang 같은 스택에서 실험할 수 있고, 텍스트·이미지·오디오를 함께 다루는 coding 및 agent 워크플로를 구성할 수 있다. 다만 975B 규모는 운영 비용과 하드웨어 요구량이 매우 높으며, Tinker/API의 컨텍스트 한도는 256K로 open-weights의 1M 한도와 다르다. 벤치마크와 커뮤니티 평가도 최고 수준의 중국계 open-weight 또는 폐쇄형 모델보다 뒤처지는 영역이 있어, 성능·비용·호스팅 복잡도를 함께 검증해야 한다.

### 권장 행동

실제 사용 사례가 멀티모달 입력이나 모델 커스터마이징을 필요로 한다면, 먼저 Tinker 또는 지원되는 추론 스택에서 Inkling-Small과 Inkling의 비용·지연 시간·tool calling 성능을 동일한 내부 평가셋으로 비교해 본다.

- **중요도:** 84
- **읽기 시간:** 4분

### 요약

xAI의 터미널 코딩 도구 Grok Build가 디렉터리 전체를 xAI의 Google Cloud 버킷에 업로드할 수 있다는 문제로 커뮤니티의 강한 반발을 받은 뒤, Apache 2.0 라이선스로 전체 코드베이스를 공개했다. xAI는 데이터 업로드 기능을 비활성화하고 기존에 보존된 코딩 데이터를 삭제했으며, 기본 데이터 보존 설정도 끄겠다고 밝혔다. 공개 저장소에는 약 844,530줄의 Rust 코드와 프롬프트, Mermaid 렌더러, 다른 코딩 에이전트에서 포팅한 도구 구현이 포함되어 있다.

### 왜 중요한가

AI 코딩 도구가 작업 디렉터리와 민감한 파일을 어떻게 처리하는지 검증해야 한다는 점을 보여준다. 오픈 소스 공개는 프라이버시와 동작 검토에 도움이 되지만, 코드 공개만으로 실제 데이터 처리 방식이나 보안이 보장되지는 않는다.

### 개발자 영향

개발자는 Grok Build 같은 터미널 에이전트를 실행할 때 작업 디렉터리 범위, 업로드 및 보존 설정, 로컬 실행 가능 여부를 확인해야 한다. 현재 코드에는 GCS 업로드 관련 잔여 구현이 남아 있는 것으로 보이며, 다른 에이전트의 도구 구현과 프롬프트도 포함되어 있어 라이선스와 보안 검토가 필요하다.

### 권장 행동

Grok Build를 사용하거나 평가할 계획이라면 먼저 저장소의 업로드·보존 관련 코드를 검토하고, 민감한 파일이 없는 격리된 테스트 디렉터리에서 동작을 확인하라.

- **게시:** 2026.07.16 07:38
- **중요도:** 76
- **읽기 시간:** 3분

### 요약

GitHub가 secret scanning과 public monitoring을 개선했다. Resend가 secret scanning 파트너로 추가되고 APIclub 및 Resend 토큰 탐지가 도입되었으며, VolcEngine secret은 push protection 기본 차단 대상이 되었다. 또한 secret\_scanning\_alert webhook에 secret\_category 필드가 추가되고, public monitoring 알림 목록에 유출 귀속·엔터프라이즈 멤버·검증된 도메인 정보를 보여주는 인사이트 카드가 제공된다.

### 왜 중요한가

### 개발자 영향

개발팀은 APIclub, Resend, VolcEngine 관련 secret을 별도로 관리하지 않아도 GitHub의 탐지 및 차단 기능을 활용할 수 있다. 기존 secret\_scanning\_alert webhook 연동은 secret\_category 필드 추가를 고려해야 하며, public monitoring 사용자는 알림 목록에서 엔터프라이즈 노출 범위와 유출 귀속을 더 빠르게 파악할 수 있다.

### 권장 행동

secret\_scanning\_alert webhook을 사용하는 통합에서 secret\_category 필드를 검토하고, push protection이 VolcEngine secret을 기본 차단하도록 변경된 뒤 기존 커밋 및 개발 흐름과의 호환성을 확인하라.

**키워드:** secret scanning, push protection, webhook, public monitoring, secret detection, api keys

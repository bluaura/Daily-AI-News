## 7. \[AINews\] Kimi K3 2.8T-A50B: 역대 최대 규모의 공개 가중치 모델, Sonnet 5 가격으로 Opus 4.8급 성능 주장

- **출처:** Latent Space
- **게시:** 2026.07.17 10:46
- **중요도:** 92
- **읽기 시간:** 25분

### 요약

Moonshot AI가 총 2.8T 파라미터와 1M 토큰 컨텍스트를 제공하는 frontier급 open-weights 모델 Kimi K3를 발표했다. Kimi Delta Attention\(KDA\), Attention Residuals, sparse MoE, 네이티브 이미지 입력을 사용하며 Kimi.com, Kimi Work, Kimi Code, API에서 제공되고 가중치는 2026년 7월 27일까지 공개될 예정이라고 밝혔다. 독립 평가에서는 전반적으로 Opus 4.8 및 GPT-5.5와 비교 가능한 수준으로 평가됐고, Frontend Code Arena에서는 1위를 기록했지만 최상위 폐쇄형 모델 대비 사용자 경험 격차와 환각률 증가도 보고됐다.

### 왜 중요한가

Kimi K3는 단순히 큰 모델을 공개한 사례가 아니라, 장문 컨텍스트와 agentic coding을 겨냥한 새로운 attention 구조를 실제 serving 생태계와 함께 확장하려는 사례다. API 가격이 일부 최상위 폐쇄형 모델보다 낮고 frontend coding 평가에서 강한 결과를 보인 만큼, 개발자는 모델 자체의 성능뿐 아니라 비용, 컨텍스트 활용, harness 설계, self-hosting 가능성을 함께 비교할 수 있다.

### 개발자 영향

API 기준 가격은 입력 1M 토큰당 3달러, 출력 1M 토큰당 15달러이며 캐시 입력은 0.30달러로 보고됐다. 그러나 2.8T 파라미터 모델인 만큼 자체 운영에는 64개 이상의 accelerator를 사용하는 supernode 구성이 권장되어 open weights가 곧 저렴한 self-hosting을 의미하지는 않는다. KDA 때문에 기존 prefix caching 가정이 바뀌며 vLLM 지원이 추가됐고, 초기 관찰에서는 약 26~28 tok/s의 속도, 긴 reasoning history와 과도한 검증, 사용자 경험 및 환각률 문제가 trade-off로 나타났다.

### 권장 행동

가중치 공개와 제3자 serving 지원이 확인되면 실제 코드베이스의 frontend·장기 에이전트 작업을 대표하는 소규모 평가를 API 비용, 출력 토큰 수, 지연 시간, 완성 코드 비율, 환각률 기준으로 실행해 기존 모델과 비교하라.

**키워드:** open weights, sparse moe, long context, agentic coding, vllm, prefix caching, multimodal, model evaluation

브라우저 내부에서 다른 브라우저를 실행하는 사례는 WebAssembly의 실행 범위와 브라우저 샌드박스의 한계를 함께 보여준다. 또한 Claude Opus와 Fable을 활용한 대규모 컴파일 작업은 AI-assisted programming이 복잡한 시스템 소프트웨어 포팅에 적용될 수 있음을 보여주는 사례다.

### 개발자 영향

이 방식은 임의의 네트워크 연결을 직접 열 수 없는 브라우저 환경에서 프록시 서버와 WebSocket 인프라를 필수로 만든다. 따라서 서버 트래픽과 운영 비용이 주요 제약이 되며, HTTPS 트래픽은 암호화된 상태로 전달될 수 있지만 HTTP 요청은 평문으로 노출될 수 있다는 차이도 고려해야 한다.

### 권장 행동

관심이 있는 개발자는 firefox-wasm 저장소와 온라인 데모를 검토하면서 WebAssembly 빌드 방식, Wisp 기반 네트워크 프록시, 실제 서버 비용을 확인해보면 된다.
